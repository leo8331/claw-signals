弹性测算 — 氧化镝涨价情景：
情景1(保守)：氧化镝维持120万 → 稀土总净利~0.8亿 → SOTP估值 9+16=25亿
情景2(中性)：氧化镝涨至300万 → 稀土净利~1.5亿 → SOTP估值 9+30=39亿
情景3(乐观)：氧化镝涨至600万 → 稀土净利~2.5亿 → SOTP估值 9+50=59亿
情景4(历史极值)：氧化镝涨至1400万 → 稀土净利~5亿 → SOTP估值 9+100=109亿

当前市值若在30-40亿 → 情景2-3之间的安全边际

📉 L3 低关注度：量化跟踪方案

🔧 方案1：百度指数爬虫（Python自动化）

原理百度指数公开的趋势折线图，通过cookie模拟登录后抓取json数据
所需工具Python + requests + 百度账号cookie
关键词"MPO光纤" / "HVLP铜箔" / "氧化镝" / "太辰光" / "德福科技" 等
输出每周搜索指数趋势变化%
难度中——需要手动获取cookie，百度有反爬
参考代码见 github.com 搜索 "baidu-index-crawler"

📱 方案2：微信指数（手动+半自动）

原理微信小程序搜索"微信指数"，输入关键词查看7/30/90天趋势
优势覆盖微信生态（公众号+视频号+搜一搜），比百度指数更反映真实讨论热度
缺点无API，需手动截图记录或OCR提取
替代方案搜狗微信搜索(sogou.com) → 搜索关键词 → 观察结果数量变化
实操每周一固定时间截图5个关键词的微信指数 → Excel记录 → 画趋势线

🐍 方案3：自建简易追踪器（推荐起步）

思路用多个公开数据源合成"关注度指数"
数据源1百度搜索结果数：site:baidu.com "关键词" → 结果数量变化
数据源2雪球热度：stock.xueqiu.com → 帖子/讨论数量
数据源3东方财富股吧：guba.eastmoney.com → 帖子量变化
数据源4Google Trends CSV导出(github.com/GeneralMills/pytrends)
合成公式L3_Score = 1 - normalized(搜索热度) → 越低越好（表示关注度低）
起步建议：先手动维护Excel → 每周花10分钟记录5个关键词的百度搜索量+雪球帖子数 → 3个月后就有足够的趋势数据判断L3变化。
这个方案的Python脚本我可以直接写给你，跑起来只需要requests和schedule两个库。

⚠️ 估值建模的局限
以上所有模型基于公开数据+机构预测。瓶颈点估值的核心困难在于：
(1) TAM数据依赖客户(NVIDIA等)的出货指引，而指引可能变化；
(2) 渗透率和市占率是主观判断，需要持续跟踪验证；
(3) 小市值公司流动性溢价/折价难以量化。

建议：每月更新一次假设，跟踪关键变量(TAM/份额/毛利率)的实际vs预期偏差。
